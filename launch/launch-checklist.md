# Launch Checklist

## Before Launch

- Rewrite the GitHub repo description to match the wedge:
  `The native red-team skill for Codex and Claude Code. Finds prompt injection, MCP poisoning, memory poisoning, and concealment bugs with forensic evidence.`
- Add a social preview image.
- Record a 30-60 second demo using `launch/demo-script.md`.
- Pin the most compelling example output in the README or repo discussion.
- Create the first GitHub release.
- Add `LICENSE` and `SECURITY.md` before broad distribution.

## Launch Day

- Publish the Show HN post from `launch/show-hn.md`.
- Publish the X thread from `launch/x-thread.md`.
- Publish the LinkedIn post from `launch/linkedin-post.md`.
- Reply fast to every serious comment in the first 2-3 hours.
- Update the README with any confusion patterns you see repeatedly.

## First 7 Days

- Ask 5-10 high-signal builders to run it on their own agent repo.
- Turn the best real finding into a follow-up post with proof.
- Publish a short retrospective: what HackYourAgent caught, what it missed, and what you changed.
- Open 3-5 good first issues so outside contributors have a path in.
- Add one more seeded vulnerable example if a recurring request appears.
